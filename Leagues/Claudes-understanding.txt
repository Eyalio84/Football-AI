  ---
  What You're Building

  A Universal League Extractor-Builder — a reusable, automated
  pipeline that can onboard any football league into Soccer-AI
  with the same depth as the existing Premier League integration.
  La Liga is the first proof-of-concept. Serie A and the Israeli
  Premier League follow. The long-term prototype extends to other
  sports (NBA, NFL, UFC, Wrestling).

  ---

  ---
  What the System Must Do (Architecture Intent)

  The extractor-builder has two outputs:
  1. Data pipeline — scouts, structures, and populates the KG for
  new clubs
  2. Integration scaffolding — generates the correct API
  endpoints, competition codes, club aliases, and fan persona
  configs so the new league slots into the existing backend with
  minimal manual work

  It should be config-driven: pass in a league definition
  (competition code, list of clubs, regional dialect map) and the
  system does the rest.

  The Reusability Vision (My Understanding)

  The system is not a one-off La Liga script. It's a factory:
                                                                    League Config (JSON)                                                  → Data Scout (API + Web + Wikipedia + CSV)
      → Knowledge Graph Builder (same node/edge schema as PL)
      → Fan Persona Generator (20 clubs × personality profile)
      → API Endpoint Registrar (competition code → live endpoints)
      → Validation Suite (schema check before integration)

  The same factory runs for Serie A with a different config. Then
  Israeli Premier League. Then, with a schema adapter, NBA or NFL.
  ---
  La Liga Specific Considerations I'd Surface Now

  - El Clásico (Real Madrid vs Barcelona) is arguably the world's
  most famous rivalry — rivalry intensity = 1.0, needs richer
  treatment than a typical derby
  - Regional identity matters deeply: Catalan (Barcelona,
  Espanyol), Basque (Athletic Bilbao, Real Sociedad), Galician
  (Deportivo), Castilian (Real Madrid, Atletico) — this maps to
  the Z-dimension (Linguistic) in the 4D persona architecture
  - La Liga uses PD as its football-data.org competition code (not
   PL)
  - The local CSV data already has La Liga matches — this is a
  significant head-start for Dixon-Coles calibration on Spanish
  fixtures
